# AI Governance Notes

An AI governance program should define the intended purpose of each system and document the operating context in which it will be used.

Governance documentation should include foreseeable risks, evaluation results, oversight responsibilities, and the controls used to reduce unacceptable risk.

Human oversight is important when an AI system affects rights, safety, access to services, employment, or other significant decisions.

Documentation should be updated when the system, data, user population, or deployment environment changes in a way that may affect performance or risk.
